What Is ECHO?
-------------
ECHO is a personal passive signals intelligence platform. It is a system designed to collect, process, analyze and visualize the invisible layer of radio frequency activity that surrounds us at all times — WiFi networks, Bluetooth devices, probe requests, and RF signals that most people never know exist.
At its core ECHO answers one question:
What is happening in the electromagnetic world around me, and what does it mean?

**************************************************************

The Two Faces of ECHO
---------------------
ECHO exists as two things simultaneously:
- The Companion — a desktop widget that lives on your screen. Always present. Always aware. Powered by a local AI personality named ECHO who processes your intelligence, narrates your data, and answers your questions. Think R2-D2 meets Alfred — a droid butler who has been running this operation long before you walked in and will be running it long after.
- The Platform — a full browser-based intelligence application. Maps, timelines, signal analysis, device profiling, behavioral pattern detection, anomaly flagging. The place where raw data becomes understanding.

**************************************************************

How ECHO Is Fed
---------------
ECHO is fed by field collection using a Marauder V7 — a purpose-built WiFi and Bluetooth analysis device. After a collection run the SD card is pulled, plugged into the host machine, and ECHO automatically detects, crawls, and ingests every capture file. No manual steps. No file sorting. ECHO handles it.
Additionally ECHO accepts historical wardriving data from WiGLE — meaning years of previous collection history can be imported on day one.

**************************************************************

The Mobile Node — MN-1
----------------------
ECHO extends into the field via the ECHO Research Node MN-1 — a 3D printed vehicle-mounted collection unit housing two Raspberry Pis working in tandem. One Pi captures. One Pi communicates. The node calls home to ECHO via a Meshtastic LoRa mesh network at defined intervals, updating the intelligence picture in real time while the operator is still in the field. All transmissions are encrypted. Nobody intercepts anything meaningful.
As more nodes are added to the fleet, each one contributes to the same shared intelligence picture. The map grows. The database deepens. ECHO gets smarter.

**************************************************************

What ECHO Does With The Data
----------------------------
Raw captures are just noise. ECHO turns noise into intelligence through five analytical lenses:
Spatial — where everything is and has ever been, on a live map.
Temporal — when things appeared, disappeared, and returned. Patterns over time.
Signal — the RF environment itself. Strength, channels, congestion, behavior.
Network Intelligence — who made the hardware, what encryption is used, what the naming patterns reveal, what is anomalous.
ECHO Intelligence — the AI layer. Natural language answers to real investigative questions drawn from the entire database.

The Science Behind It
---------------------
ECHO is not just a collection tool. It is a data science platform. Every device it has ever seen has a profile. Every profile has a behavioral model. The system learns what normal looks like in your environment and surfaces what doesn't fit. Over time ECHO builds a predictive understanding of the RF world around you — flagging the device that shouldn't be there before you think to look for it.

## But Before That — The PMKID Distinction

You described EAPOL handshake capture perfectly. PMKID is actually slightly different and worth knowing because it's more elegant and more dangerous.

Traditional handshake capture requires waiting for or forcing a client to reconnect. You need a victim device present.

PMKID capture requires only the **access point.** No client needed. No deauth needed. No waiting.

The PMKID is a value the router broadcasts that is mathematically derived from the network password, the router's MAC, and the client MAC. Capture it from the router alone and you have everything needed to attempt offline cracking.

```
┌─────────────────────────────────────────┐
│  TRADITIONAL vs PMKID                   │
│                                         │
│  Traditional:                           │
│  Need: AP + Client device present       │
│  Method: Wait or force reconnect        │
│  File: .cap / .pcap                     │
│                                         │
│  PMKID:                                 │
│  Need: AP only                          │
│  Method: Single frame request to AP     │
│  File: .pmkid / .cap                    │
│                                         │
│  Same end result — offline crack        │
│  attempt against network password       │
│  PMKID just removes the dependency      │
│  on a client being present              │
└─────────────────────────────────────────┘
```

## The Bluetooth Capture Modes

You understand WiFi captures. Bluetooth follows the same passive listening principle but the targets are different.

Remember — Bluetooth tells you about **people** not places.

---

**MODE 1 — Bluetooth Wardrive**

```
┌─────────────────────────────────────────┐
│  BLUETOOTH WARDRIVE                     │
│                                         │
│  [Marauder] ←── listens ──[BT Devices] │
│       │                                 │
│       │ records                         │
│       ▼                                 │
│  MAC, Device Name, Device Type,        │
│  RSSI, GPS, Timestamp                  │
│       │                                 │
│       ▼                                 │
│  bt_wardrive_[timestamp].csv           │
└─────────────────────────────────────────┘
```

Mobile sweep capturing every broadcasting Bluetooth device in range. Phones, headphones, watches, speakers, medical devices, car systems. All of them.

Every device that has Bluetooth enabled and is in discoverable or active state gets logged with its GPS location.

---

**MODE 2 — BLE Sniff & Analyzer**

BLE means Bluetooth Low Energy. This is the protocol used by fitness trackers, smartwatches, medical devices, IoT sensors, AirTags, and most modern wearables.

Different protocol than classic Bluetooth. Lower power. Designed for devices that need to broadcast small amounts of data continuously for months on a battery.

```
┌─────────────────────────────────────────┐
│  CLASSIC BLUETOOTH vs BLE               │
│                                         │
│  Classic:                               │
│  Higher power │ Richer data             │
│  Phones, speakers, car systems          │
│  Range ~10m                             │
│                                         │
│  BLE:                                   │
│  Ultra low power │ Small data packets   │
│  Wearables, trackers, sensors           │
│  Range ~50m in open space               │
│                                         │
│  ECHO captures both separately          │
│  because they tell different stories    │
└─────────────────────────────────────────┘
```

---

**MODE 3 — AirTag Detection**

Apple AirTags and similar tracking devices use BLE to broadcast their presence. The Marauder can detect these broadcasts passively.

For ECHO this is a defensive capability. Detecting unwanted tracking devices near a person or vehicle. Also intelligence relevant — an AirTag in an unexpected location tells a story.

---

**MODE 4 — Flipper Zero Detection**

The Flipper Zero is a multi-tool security device similar in spirit to the Marauder. It has a distinctive BLE signature when active. The Marauder can detect it.

For ECHO this means knowing when another active collection or testing device is operating in your environment. Operational awareness.

---

**MODE 5 — Card Skimmer Detection**

Bluetooth card skimmers installed on ATMs and gas pumps broadcast via Bluetooth to allow criminals to retrieve stolen data wirelessly. They have recognizable device naming patterns and broadcast signatures.

The Marauder detects these passively. ECHO flags them automatically when ingesting Bluetooth captures.

This is one of the clearest defensive use cases you can demonstrate to a corporate client.

---

**MODE 6 — Flock & Meta Detect**

Meta devices — Ray-Ban smart glasses, Meta Portal devices — have identifiable BLE signatures. Flock detection identifies clusters of devices moving together suggesting a group.
